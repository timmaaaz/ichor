3. NewDatabase [dbtest infrastructure]
   - Location: business/sdk/dbtest/dbtest.go
   - Function: NewDatabase(t *testing.T, testName string) *Database
   - Returns:
     * DB: isolated *sqlx.DB
     * Log: *logger.Logger
     * BusDomain: all instantiated business packages
   - Properties:
     * DB name: random 4 lowercase letters
     * TimeZone: set to 'America/New_York'
     * Each test gets isolated database instance
     * Runs migrations + seeds before returning
   - Two seeding entry points: InsertSeedData (live) + NewDatabase (test)
   - Both use same BusDomain struct and seed chain

4. BusDomain [dbtest infrastructure]
   - Central struct holding all instantiated business packages
   - Passed into all seed functions
   - Includes 60+ bus instances (Delegate, geography, HR, core, assets, products, procurement, inventory, sales, config, workflow)
   - Single source of truth for all seeded entities

5. SeedOrchestrator [dbtest]
   - Location: business/sdk/dbtest/seedFrontend.go (~71 lines)
   - Entry: InsertSeedData(log, cfg sqldb.Config) error
   - Execution order (FIXED — do not reorder):
     1. seedFoundation(ctx, busDomain)
     2. seedGeographyHR(ctx, busDomain)
     3. seedAssets(ctx, busDomain, foundation)
     4. seedProducts(ctx, busDomain, geoHR, foundation)
     5. seedInventory(ctx, busDomain, foundation, geoHR, products)
     6. seedSales(ctx, busDomain, foundation, geoHR, products)
     7. seedProcurement(ctx, busDomain, foundation, geoHR, products, inventory)
     8. seedTableBuilder(ctx, busDomain, adminID)
     9. seedPages(ctx, log, busDomain)
     10. seedForms(ctx, log, busDomain)
     11. seedWorkflow(ctx, log, busDomain, adminID)
   - Dependency rule: each function only references IDs from prior results, never reach into busDomain

6. Seed Functions (11 Total)

   seedFoundation:
     - Returns: FoundationSeed (Admins, Reporters, Bosses, USDCurrencyID, Currencies)
     - Seeds: users, roles, user roles, table access, currencies
     - adminID extracted from foundation.Admins[0].ID for later use

   seedGeographyHR:
     - Returns: GeographyHRSeed (Cities, Streets, ContactInfos, Offices)
     - Seeds: cities, streets, contact infos, offices, titles, reports-to, approval comments

   seedAssets:
     - Takes: foundation dependency
     - Seeds: asset types, conditions, valid assets, assets, approval/fulfillment statuses, tags, user assets

   seedProducts:
     - Takes: geoHR, foundation dependencies
     - Returns: ProductsSeed
     - Seeds: brands, categories, products, costs, physical attributes, metrics, cost histories

   seedInventory:
     - Takes: foundation, geoHR, products dependencies
     - Returns: InventorySeed
     - Seeds: warehouses, zones, locations, inventory items, lot trackings, serials, inspections, transfers, transactions, adjustments

   seedSales:
     - Takes: foundation, geoHR, products dependencies
     - Seeds: customers, order fulfillment statuses, orders, line item fulfillment statuses, order line items

   seedProcurement:
     - Takes: foundation, geoHR, products, inventory dependencies
     - Seeds: suppliers, supplier products, PO statuses, POs, PO line items

   seedTableBuilder:
     - Takes: busDomain, adminID
     - Seeds: 20+ module table configs (static from seedmodels/tables_*.go)

   seedPages:
     - Takes: log, busDomain
     - Seeds: page configs, content blocks, action buttons, nav pages, role-page access
     - References seedmodels/pageactions.go, seedmodels/pages.go

   seedForms:
     - Takes: log, busDomain
     - Key fact: uses busDomain.Workflow.QueryEntityByName (read-only lookup)
     - Must NOT run after seedWorkflow
     - Seeds: forms and all form fields
     - References seedmodels/tableforms.go, seedmodels/forms.go, seedmodels/formregistry.go

   seedWorkflow:
     - Takes: log, busDomain, adminID
     - Must run LAST (depends on forms already seeded)
     - Seeds: 15 action templates + 8 default automation rules

8. Permission Seeding (Three Layers)

   Layer 1: Page access (router → canAccessPage)
     - File: business/sdk/dbtest/seedmodels/pages.go → AllPages slice
     - Behavior: seedPages() auto-grants can_access=true to ZZZADMIN for every AllPages entry
     - Failure mode: missing → router redirects to /login

   Layer 2: Table access (API auth middleware)
     - File: business/sdk/migrate/sql/seed.sql → core.table_access
     - Format: (gen_random_uuid(), ZZZADMIN_id, 'schema.table', true, true, true, true)
     - Failure mode: 401 "user does not have permission READ for table: ..."

   Layer 3: Action permissions (workflow)
     - File: business/sdk/migrate/sql/seed.sql → workflow.action_permissions
     - Failure mode: 403 "permission_denied" on POST /v1/workflow/actions/{type}/execute

9. TestSeed Functions Pattern [bus packages]
   - Location: business/domain/{area}/{entity}bus/testutil.go (NOT in test files)
   - Signature: func TestSeed{Entities}(ctx context.Context, n int, ...depIDs, api *Business) ([]Entity, error)
   - Imported by integration tests in api/cmd/services/ichor/tests/{area}/{entity}api/seed_test.go
   - Lives in bus package (not test file) to avoid circular imports

EXTENSION PATTERNS:

To add new domain to seeding:
  1. Add bus field(s) to BusDomain struct (dbtest.go)
  2. Instantiate in newBusDomains function
  3. Create business/sdk/dbtest/seed_<domain>.go with seed<Domain>() function
  4. Add call in dependency chain (seedFrontend.go) at correct position
  5. Add seedmodels/tables_<domain>.go if static configs needed

To add TestSeed* function:
  1. Create function in business/domain/{area}/{entity}bus/testutil.go (NOT in test file)
  2. Call from insertSeedData in integration tests
  3. Lives in bus package to avoid circular imports
  4. Signature: TestSeed{Entities}(ctx, n int, ...depIDs, api *Business) ([]Entity, error)

To change seed ordering:
  1. Update orchestrator chain in seedFrontend.go
  2. Dependency rule: function can only reference IDs from prior result structs
  3. Never reach into busDomain for IDs that should come from earlier results
